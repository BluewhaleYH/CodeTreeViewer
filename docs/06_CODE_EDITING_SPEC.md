# 06. 코드 편집 (CODE EDITING) SPEC

> 앱 내 코드 편집기, 저장, 재분석 트리거, diff 하이라이트, 영향 범위, 편집 안전장치를 정의한다.
> **단계: 2차** (`00` §8.1). 분석 엔진은 `02`, 시각화 동기화는 `03` §10.

## 1. 범위 / 단계

- 본 기능군은 전체가 **2차** 범위다. MVP에서 미리 구현하지 않는다. (`00` §8)
- MVP 단계의 그래프↔코드 연동은 **읽기 전용 보기/이동**까지만(편집 없음). (`03` §10)

## 2. 편집기

- **Monaco Editor**를 사용한다. (`00` §4)
- 그래프 노드 → 소스 파일을 편집기로 열어 해당 위치로 이동한다. (`03` §10)
- 구문 강조, 라인 점프, 다중 파일 탭/세션 연동(상세 `추가)`).

## 3. 저장

- 편집 내용을 디스크에 저장한다. (사용자 프로젝트 파일을 직접 수정 — `01` §6의 "쓰지 않음" 원칙의 예외, 명시적 편집 시에만)
- 저장 전/후 안전장치는 §6.

## 4. 재분석 트리거

- 파일 저장(또는 명시적 재분석) 시 **변경 파일 중심으로 증분 재분석**한다. (`02` §3)
- 재분석 후 그래프/검색 인덱스를 갱신한다. (`02` §7, `05` §5)

## 5. diff 하이라이트 / 영향 범위

- 편집 전/후 **diff를 하이라이트**한다(Monaco diff).
- 재분석 결과로 **영향 범위**(추가/삭제/변경된 노드·엣지)를 그래프에서 표시한다.
- 그래프 전/후 비교 모드는 **추후** 범위. (`00` §8.1)

## 6. 편집 안전장치

- 외부에서 파일이 변경된 경우(앱 밖 수정) 충돌을 감지하고 덮어쓰기 전에 경고한다.
- 저장은 원자적 쓰기를 사용한다. (`01` §10 정책 준용)
- 미저장 변경이 있는 상태로 탭/앱 종료 시 경고한다.
- 되돌리기/다시하기는 편집기 기본 기능을 따른다.

## 7. 성공 기준(초안)

- **F-E1.** 노드에서 소스를 열어 편집·저장할 수 있다.
- **F-E2.** 저장 시 증분 재분석이 돌고 그래프/검색이 갱신된다.
- **F-E3.** 편집 전/후 diff와 영향 범위가 표시된다.
- **F-E4.** 외부 변경/미저장 종료 등에서 데이터 손실 없이 경고한다.

> 확정 전 사용자와 합의. (`00` §9)

## 추가) — 미해결 / 제안

- **(편집-1)** 편집 대상 범위(분석 대상 언어 파일만 vs 임의 텍스트 파일).
- **(편집-2)** 증분 재분석의 트리거 시점(저장 시 자동 vs 수동 버튼).
- **(편집-3)** 영향 범위 시각화 방식(색/배지/별도 패널).

- **(편집-5)** 멀티 파일 미저장 상태의 세션 복원 처리(`01` §5와 연계).
