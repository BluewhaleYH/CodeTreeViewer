# 05. 검색 (SEARCH) SPEC

> 코드 검색(파일/함수명)과 로그 검색, 매칭 규칙, 퍼지, 인덱싱, 히스토리를 정의한다.
> 코드 모델은 `02`, 시각화 연동은 `03` §10, 로그 분석은 `04`.

## 1. 범위 / 단계

- **MVP**: **파일/함수명 기본 검색**. (`00` §8.1)
- **2차**: **로그 검색**(`04` 연동).

## 2. 코드 검색 (MVP)

- **파일, 함수 검색 기능**을 제공한다. (TEMP_SPEC)
- 검색 대상: 분석 산출물의 노드 — **파일명 + 함수/메서드 정의명**(경로 포함). (`02` §4)
  - MVP에서 함수 **정의**는 추출되므로(`02` §1, §4.1) 함수명 검색이 MVP에서 동작한다. (함수 호출 그래프와 무관)
- 검색 결과 선택 → **그래프 포커스/노드 선택 동기화**. (`03` §10)
- 현재 활성 탭(프로젝트) 범위에서 검색한다. 탭 간 교차 검색은 `추가)`.

## 3. 매칭 규칙

- 기본: 대소문자 무시 부분 일치.
- 옵션(설계 후보): 대소문자 구분, 정확 일치, 경로 포함 검색.
- 결과 정렬: 일치 위치/길이 기반 점수 + 이름 사전순(상세 `추가)`).

## 4. 퍼지 검색

- 파일/함수명 대상 **퍼지 매칭**을 제공한다(오탈자·부분 토큰 허용).
- 라이브러리는 OSS 우선(`00` §4). 후보는 `추가)`.

## 5. 인덱싱

- 노드 이름 인덱스를 구축해 즉시 검색을 지원한다.
- 인덱스는 분석 캐시(`02` §7)와 함께 관리하며 재분석 시 갱신한다.

## 6. 로그 검색 (2차)

- 로그 덤프 내 텍스트/정규식 검색. (`04` §8)
- 대용량 로그 대응(가상 스크롤·인덱스). (`04` §3)
- 검색 결과 ↔ 로그 라인 ↔ 그래프/코드 3중 연동. (`04` §7)

## 7. 검색 히스토리

- 최근 검색어를 보관하고 재사용할 수 있게 한다.
- 보관 범위(탭별/전역)·개수·영속성 여부는 `추가)`.

## 8. 성공 기준(초안)

- **E-S1.** 파일/함수명으로 검색하면 일치 노드가 즉시 나열된다.
- **E-S2.** 결과 선택 시 그래프가 해당 노드로 포커스된다. (`03` §10)
- **E-S3.** 퍼지 검색이 부분/오탈자 입력에도 합리적 결과를 낸다.

> 확정 전 사용자와 합의. (`00` §9)

## 추가) — 미해결 / 제안

- **(검색-1)** 퍼지 검색 라이브러리 선택(Fuse.js 등) 및 가중치/임계값.
- **(검색-2)** 검색 범위 토글(현재 탭 vs 전체 탭, 외부 노드 포함 여부).
- **(검색-3)** 검색 히스토리 영속성/공유 범위.
- **(검색-4)** 함수 검색 결과의 식별 표기(동명 함수 구분: 클래스/시그니처).
