# 03. 시각화 (VISUALIZATION) SPEC

> 화면 레이아웃, 관계도(태양계형)/트리, 뷰 전환, 인터랙션, 색상, 대규모 렌더링, 선택/하이라이트, 테마, 그래프↔코드 동기화를 정의한다.
> 데이터 모델은 `02` §4, 검색은 `05`, 로그 연동은 `04`.

## 1. 범위 / 단계

- **MVP**: 관계도/트리 시각화, 뷰 전환, 파일 클릭 시 부모/자식 표시, 영역 색상.
- **2차**: 함수 클릭 시 호출처 표시(`02` §6), 그래프↔코드 동기화 심화.
- **추후**: 그래프 전/후 비교 모드, 라이트 모드(§9).

## 2. 화면 레이아웃

- 기본 구성: **탭 바(상단) + 그래프 캔버스(주 영역) + 보조 패널(검색/노드 정보)**.
- **코드의 구조·관계를 나타내는 시각화는 오직 관계도 또는 트리 형태로만 표현한다.** 막대/원형 차트 등 다른 시각화 형식은 사용하지 않는다. (TEMP_SPEC, `00` §10 D8)
  - 단, **로그 뷰·코드 뷰**(텍스트 패널, `04` §6·`06`)는 "관계 시각화"가 아니라 보조 영역이므로 본 제약의 대상이 아니다. 이들에서 도출되는 **관계 표현**은 그래프 캔버스(관계도/트리)로만 그린다.
- 로그 분석 활성 시 3-뷰 레이아웃으로 전환된다. (`04` §6)

## 3. 관계도(태양계형) 뷰

- 선택 노드를 **중심**에 두고, 연결된 노드를 **방사형(태양계처럼)** 으로 배치한다. (TEMP_SPEC)
- 부모/자식 방향성은 `02` §4.3을 따른다.
- 거리/궤도는 관계의 깊이를 표현(중심에서 멀수록 간접 관계).

## 4. 트리 뷰

- 계층 구조를 위→아래(또는 좌→우)로 펼친다.
- 노드 접기/펼치기를 지원한다.

## 5. 뷰 전환 / 인터랙션

### 5.1 초기 뷰 (프로젝트 로드 시)

- 프로젝트를 새로 열어 분석이 끝나면 기본 뷰를 표시한다.
- 기본 표시 정책(제안, 정확한 기본값은 `추가)`에서 확정):
  - **소규모**: 전체 의존성 그래프 개요를 표시.
  - **대규모**: 진입점/루트(또는 군집) 중심 개요를 표시하고 클릭으로 점진적 확장. (`§8`)
- 세션 복원으로 열린 경우에는 **마지막 선택 노드/뷰 모드**로 복원한다. (`01` §5)

### 5.2 뷰 전환

- **관계도 ↔ 트리 전환**이 가능하다. (TEMP_SPEC) 전환해도 선택 노드/포커스는 유지한다.

### 5.3 노드 인터랙션

- **파일 노드 클릭**: 해당 파일을 중심으로 **부모·자식 관계**를 표시한다. (TEMP_SPEC)
  - 기본 동작: **직접 이웃(1-홉) 부모/자식**을 우선 표시하고, 더블클릭/확장으로 더 깊은 관계를 펼친다. (기본 표시 홉 수는 `추가)`)
- **함수 노드 클릭(2차)**: 해당 함수를 마지막 노드로 하여 **호출하는 곳들(호출처)** 을 역방향으로 표시한다. (TEMP_SPEC, `02` §6)
- 팬/줌, 노드 드래그, 더블클릭 확장 등 기본 그래프 조작을 제공한다.

## 6. 색상 / 영역 구분

- 프로젝트의 **영역을 색으로 구분**하여 표시한다. (TEMP_SPEC)
- 영역 분류는 `02` §5를 따른다(**사용자 정의 매핑 + 안드로이드 기본 프리셋**).
- **안드로이드 프리셋 기본 팔레트**(TEMP_SPEC 예시를 기본값으로 고정. 다크 테마 기준):

  | 영역 | 기본 색 |
  | ---- | ------ |
  | Application | 주황 |
  | Framework | 초록 |
  | Stack | 파랑 |
  | unknown(분류 불가) | 중립(회색) |

- 사용자가 영역→색 매핑을 변경할 수 있다(편집 UI는 `추가)`). 정확한 색상값(hex)은 디자인 단계에서 확정.
- **범례(legend)** 를 제공하며, 현재 적용된 매핑/프리셋의 영역을 반영한다.
- 색맹 접근성 대비 팔레트는 `추가)`.

## 7. 렌더링 엔진

- Canvas/WebGL 기반. 후보: **Cytoscape.js / sigma.js**. (`00` §4) — **최종 선택 미정**(`추가)`).
- 레이아웃 알고리즘(방사형/계층형)은 선택 라이브러리의 레이아웃을 활용하되, 태양계형은 커스텀 배치가 필요할 수 있다.

## 8. 대규모 렌더링

- 수천~수만 노드에서도 탐색 가능해야 한다. (`00` §6)
- 기법(설계 후보): 인접 N-홉만 렌더, 레벨 오브 디테일(LOD), 가상화, 점진적 로딩, 군집(clustering).
- 성능 합격선(노드 수/프레임률)은 **미확정** — 사용자와 확정. (`추가)`, `00` §9)

## 9. 선택 / 하이라이트 / 테마

- 선택 노드와 그 관계 경로를 하이라이트하고 나머지는 디엠퍼사이즈한다.
- **테마**: 기본 **다크**(`00` §10 D3). **라이트 모드는 추후**(`00` §8.1).

## 10. 그래프 ↔ 코드 동기화

- 노드 선택 시 노드 정보(경로/심볼/영역)를 보조 패널에 표시한다.
- 노드 → 소스 위치로 이동(파일/라인). 편집기 연동은 `06`(2차).
- 검색 결과 → 그래프 포커스 동기화는 `05`.
- 로그 라인 ↔ 노드 ↔ 코드 3중 연동은 `04` §7(2차).

## 11. 성공 기준(초안)

- **C-V1.** 파일 클릭 시 부모/자식 관계가 관계도·트리 양쪽에서 올바르게 그려진다.
- **C-V2.** 관계도↔트리 전환 시 선택/포커스가 유지된다.
- **C-V3.** 영역별 색상이 일관되게 적용되고 범례로 식별된다.
- **C-V4.** 대규모 그래프에서도 멈춤 없이 탐색 가능(합격선 확정 후 측정).
- **C-V5.** 관계/구조 시각화가 관계도·트리 외 형식으로 표시되지 않는다. (`00` §10 D8)

> 확정 전 사용자와 합의. (`00` §9)

## 추가) — 미해결 / 제안

- **(시각화-1)** 렌더링 라이브러리 최종 선택(Cytoscape.js vs sigma.js): 대규모 성능·태양계 커스텀 배치 용이성 비교 필요.
- **(시각화-2)** 대규모 렌더링 성능 합격선(노드 수, 목표 FPS, 초기 표시 홉 수).
- **(시각화-3)** 외부/미해결 노드(`02` §9)의 표시 토글 UI.
- **(시각화-4)** 태양계형 배치의 정확한 규칙(궤도 간격, 다중 부모 처리, 사이클 처리).
- **(시각화-5)** 색맹 접근성 팔레트 / 영역 색 사용자 커스터마이즈(영역→색 매핑 편집 UI). 분류 *방식*은 `02` §5에서 확정됨.
- **(시각화-6)** 노드 라벨 밀집 시 표시 정책(줌 레벨별 라벨 노출).
- **(시각화-7)** 초기 뷰(§5.1) 기본값: 소/대규모 경계 기준, 대규모 시 진입점/루트 선정 규칙(군집 기준 포함).
- **(시각화-8)** 파일 클릭 시 기본 표시 홉 수(1-홉 고정 vs 사용자 설정) 및 확장 동작 세부.
