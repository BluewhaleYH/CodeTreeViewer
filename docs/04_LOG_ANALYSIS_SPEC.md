# 04. 로그 덤프 분석 (LOG ANALYSIS) SPEC

> 로그 덤프 파일 열람, 로그→코드 역추적, 대용량 처리, 필터링, 3-뷰 레이아웃, 3중 연동을 정의한다.
> **단계: 2차** (`00` §8.1). 코드 분석은 `02`, 시각화는 `03`, 검색은 `05`.

## 1. 범위 / 단계

- 본 기능군은 전체가 **2차** 범위다. MVP에서 미리 구현하지 않는다. (`00` §8)
- 추후: 로그 매칭 신뢰도 표시(§5.3).

## 2. 로그 덤프 열람

- 로그들이 저장된 **덤프 파일을 열어볼 수 있는 영역**을 제공한다. (TEMP_SPEC)
- 텍스트 로그 파일을 로드해 라인 단위로 표시한다.
- 라인 번호, 가로 스크롤, 줄바꿈 토글 등 기본 열람 기능.

## 3. 대용량 처리

- 대용량 로그를 가정한다(수십만~수백만 라인).
- 기법(설계 후보): 스트리밍 로드, 가상 스크롤(보이는 영역만 렌더), 인덱스 기반 탐색.
- 전체를 메모리에 올리지 않고도 검색·점프가 가능해야 한다. (`05` 연동)

## 4. 필터링

- 레벨/태그/텍스트/정규식 기준 필터를 제공한다(구체 항목은 `추가)`).
- 필터는 표시에만 적용하고 원본은 보존한다.

## 5. 로그 → 코드 역추적

- 특정 **로그 라인을 선택**하면 **그 로그를 출력하는 코드 위치를 프로젝트에서 탐색해 표시**한다. (TEMP_SPEC)
- 또한 **해당 파일/함수의 관계를 시각적으로 함께 표시**한다. (TEMP_SPEC, `03` §5)

### 5.1 매칭 방법(설계 후보)

- 로그 메시지 → 소스의 로그 호출 문자열 패턴 매칭(예: 포맷 문자열, 태그).
- 정적 문자열은 직접 매칭, 포맷/가변 부분은 패턴화하여 후보를 산출.

### 5.2 다중 후보 처리

- 매칭 후보가 여러 개면 후보 목록을 제시하고 사용자가 선택한다.

### 5.3 신뢰도 표시(추후)

- 매칭 신뢰도(점수)를 표시하는 기능은 **추후** 범위. (`00` §8.1)

## 6. 3-뷰 레이아웃

- 로그 분석 활성 시 화면을 **3-뷰**로 구성한다(초안):
  1. **로그 뷰** — 덤프 라인 목록/필터
  2. **그래프 뷰** — 관계도/트리 (`03`)
  3. **코드 뷰** — 매칭된 소스 위치 (`06` 편집기 연동, 2차)
- 레이아웃 비율/배치 상세는 `추가)`.

## 7. 3중 연동

- **로그 라인 ↔ 그래프 노드 ↔ 코드 위치**가 서로 연동된다.
  - 로그 라인 선택 → 해당 코드 노드 그래프 포커스 + 코드 뷰 이동.
  - 그래프 노드 선택 → 관련 로그 라인 하이라이트(가능한 경우).
- 동기화 규칙 상세는 `03` §10과 함께 정의.

## 8. 로그 검색

- 로그 내 검색을 제공한다. (TEMP_SPEC) 규칙/인덱싱은 `05`.

## 9. 성공 기준(초안)

- **D-L1.** 덤프 파일을 열어 라인 단위로 열람·스크롤할 수 있다.
- **D-L2.** 로그 라인 선택 시 해당 출력 코드 위치를 찾아 그래프·코드와 함께 표시한다.
- **D-L3.** 매칭 후보가 여러 개면 오탐을 단정하지 않고 후보를 제시한다. (`00` §9 C1)
- **D-L4.** 대용량 로그에서도 멈춤 없이 열람·검색된다.

> 확정 전 사용자와 합의. (`00` §9)

## 추가) — 미해결 / 제안

- **(로그-1)** 지원 로그 포맷(안드로이드 logcat, 커널 dmesg, 임의 텍스트 등)과 파서 범위.
- **(로그-2)** 로그→코드 매칭 알고리즘의 정확도 기준과 포맷 문자열 역매칭 방식.
- **(로그-3)** 3-뷰 레이아웃의 기본 배치/리사이즈/숨김 동작.
- **(로그-4)** 필터 항목 표준(레벨/태그/PID/TID/시간 범위 등).
- **(로그-5)** 로그 인덱싱을 영구 캐시할지 세션 한정으로 둘지.
