# 01. 앱 환경 (APP ENVIRONMENT) SPEC

> 앱의 실행 형태, 창 제어, 다중 탭, 상태 영속성, 저장 위치, 단일 인스턴스, 업데이트, 세션 손상 대비를 정의한다.
> 상위 개요: `00_OVERVIEW_SPEC.md`. 배포 상세: `docs/DEPLOY.md`.

## 1. 범위

- 본 문서는 **셸/플랫폼 레벨** 요구사항을 다룬다. 화면 안의 시각화는 `03`, 분석은 `02`.
- MVP 범위: §2~§5(실행 형태·창·다중 탭·상태 영속성), §7(단일 인스턴스), §10(세션 손상 대비).
- §9(자동 업데이트)는 배포 단계에서 활성화하되 설계는 MVP에서 함께 고려.

## 2. 실행 형태 / 플랫폼

- 브라우저에서 띄우는 형태가 아닌, **Windows / Ubuntu(Linux)에서 동작하는 독립 실행 프로그램**이어야 한다. (TEMP_SPEC)
- Electron + TypeScript 기반. (`00` §4)
- 개발 실행 시 Linux의 `chrome-sandbox` SUID 문제를 피하려고 `dev`/`start`에 한해 샌드박스를 끈다(`ELECTRON_DISABLE_SANDBOX=1`). **패키징 배포본은 정상 샌드박스로 동작**한다. (CLAUDE.md §9)

## 3. 창(Window) 제어

- **최소화 / 최대화 / 종료** 기능을 제공한다. (TEMP_SPEC)
- 창 크기·위치·최대화 여부는 세션 상태로 저장·복원한다. (§5)
- 멀티 모니터 환경에서 마지막 위치가 화면 밖이면 보이는 영역으로 보정한다.

## 4. 다중 탭 / 프로젝트 선택

- 사용자가 **컴퓨터 내 프로젝트 폴더를 직접 선택**한다. (TEMP_SPEC)
- **여러 개의 탭**이 존재하며 **각 탭마다 다른 프로젝트를 선택**할 수 있다. (TEMP_SPEC)
- 탭 = 프로젝트 1개. 탭별로 독립적인 분석 결과·뷰 상태를 가진다.
- 탭 추가/닫기/전환을 지원한다. 닫은 탭의 복원 여부는 `추가)`.

## 5. 상태 영속성(세션)

- 프로그램이 종료되어도 **마지막에 띄웠던 프로젝트/파일이 유지**되어야 한다. (TEMP_SPEC)
- 복원 대상(MVP):
  - **열린 탭 목록**과 각 탭의 **프로젝트 경로**
  - 각 탭의 **열린 파일(선택/포커스 노드)**
  - 각 탭의 **뷰 모드**(관계도/트리) (`03` §3,§4)
  - 활성 탭, 창 크기·위치·최대화 상태(§3)
- 저장 시점: 상태 변경 시(디바운스) + 정상 종료 시. 비정상 종료 대비는 §10.

## 6. 저장 위치

- 세션/설정은 Electron의 OS별 사용자 데이터 디렉터리(`app.getPath('userData')`)에 저장한다.
- 분석 캐시 저장 위치·정책은 `02` §7에서 정의한다(세션 상태와 분리).
- 사용자 프로젝트 폴더 내부에는 기본적으로 파일을 쓰지 않는다. (읽기 위주)

## 7. 단일 인스턴스

- 앱은 **단일 인스턴스**로 동작한다. 두 번째 실행 시 기존 창을 포커스한다.
- (구현: Electron `requestSingleInstanceLock`)

## 8. 메뉴 / 단축키

- 최소 메뉴(파일): 프로젝트 열기, 로그 덤프 열기, 새 탭, 탭 닫기, 닫은 탭 다시 열기, 종료.
- **수정자 키는 Ctrl 기준**(주 대상 Windows/Linux, macOS는 배포 대상 아님). (`00` §10 D13)
- **전체 단축키 표(확정)**:

  | 단축키 | 동작 | 구현 |
  | ---- | ---- | ---- |
  | `Ctrl+O` | 프로젝트 열기(폴더 선택) | 메뉴 |
  | `Ctrl+L` | 로그 덤프 열기 | 메뉴 |
  | `Ctrl+T` | 새 탭 | 메뉴 |
  | `Ctrl+W` | 탭 닫기(미저장 시 경고) | 메뉴 |
  | `Ctrl+Shift+T` | 닫은 탭 다시 열기(최근 순) | 메뉴 |
  | `Ctrl+1`~`9` | N번째 탭으로 전환(`9`=마지막) | 렌더러 |
  | `Ctrl+Shift+G` | 관계도 ↔ 트리 뷰 전환 | 렌더러 |
  | `Ctrl+F` | 검색 입력 포커스 | 렌더러 |
  | `Ctrl+S` | 편집 중 소스 저장 | 편집기(Monaco) |

- 메뉴 가속기(Ctrl+O/L/T/W/Shift+T)와 렌더러 단축키(Ctrl+1~9·Shift+G·F)는 키가 겹치지 않아 충돌하지 않는다.

## 9. 업데이트(자동 배포·업데이트)

- 무서명 자동 배포·자동 업데이트를 목표로 한다. (CLAUDE.md §9, `docs/DEPLOY.md`)
- Linux: AppImage + deb / Windows: 포터블 zip, nsis 설치본 + zip.
- 업데이트 채널·주기·롤백 정책은 `docs/DEPLOY.md`에서 상세화(본 문서는 요구사항만 명시).

## 10. 세션 손상 대비

- 세션 파일이 손상/부분기록된 경우에도 앱은 **기동에 실패하지 않는다.**
- 정책:
  - 세션 파일은 **원자적 쓰기**(임시 파일 → rename)로 저장한다.
  - 로드 시 파싱 실패하면 **직전 백업본**으로 폴백하고, 그것도 실패하면 빈 세션으로 안전 기동한다.
  - 손상 감지 시 사용자에게 알림(비차단)을 표시한다.

## 11. 성공 기준(초안)

- **E1.** 폴더 선택 → 새 탭에 프로젝트가 로드된다.
- **E2.** 앱 재시작 시 탭/열린 파일/뷰 모드/창 상태가 복원된다. (`00` §9 C2)
- **E3.** 강제 종료(프로세스 kill) 후 재기동 시 마지막(또는 직전) 세션으로 복원되며 크래시하지 않는다.
- **E4.** 두 번째 실행 시 새 창이 아니라 기존 창이 포커스된다.

> 확정 전 사용자와 합의. (`00` §9)
